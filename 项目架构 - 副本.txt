GH5 项目架构文档
================================================================================

一、项目概述
================================================================================
GH5 是一个纯前端静态产品展示网站，使用 GitHub 作为数据存储后端。
支持多语言切换、产品筛选、对比、浏览历史等功能。

三、HTML 页面说明
================================================================================

3.1 index.html（主站首页）
--------------------------------------------------------------------------------
功能：产品展示、多语言切换、搜索、筛选、对比、浏览历史
包含模块：
  - 轮播图 (carousel)
  - 搜索栏 (product-search)
  - 产品列表 (product-series)
  - 联系方式表单 (contact-section)
  - 页脚 (footer)

3.2 admin.html（功能设置）
--------------------------------------------------------------------------------
功能：控制网站各功能的开启/关闭
包含开关：
  - toggle-backtotop     回到顶部
  - toggle-imagezoom     图片放大
  - toggle-search        产品搜索
  - toggle-compare       产品对比
  - toggle-history       浏览历史
  - toggle-filter        产品筛选
  - toggle-layout        布局切换
  - toggle-share         社交分享
  - toggle-theme         主题切换
  - toggle-language      语言切换
  - toggle-contact       联系方式

3.3 i18n-admin.html（翻译管理）
--------------------------------------------------------------------------------
功能：管理界面翻译文本
功能：
  - 查看所有翻译 key
  - 导出翻译为 JSON
  - 导入翻译 JSON 文件

3.4 config/config.html（后台管理）
--------------------------------------------------------------------------------
功能：产品管理、图片上传、系列管理
子页面（通过 hash 路由）：
  - #product-editor    产品编辑
  - #upload           图片上传
  - #series-manager   系列管理
  - #config-editor    网站配置
  - #translation-manager 翻译管理

六、产品图片命名规则
================================================================================

文件名格式：产品名 (序号).扩展名

示例：
  产品1 (1).jpg   → 产品"产品1"的主图
  产品1 (2).jpg   → 产品"产品1"的详情图1
  产品1 (3).jpg   → 产品"产品1"的详情图2

自动分组逻辑：
  - 提取文件名中的"产品名"和"序号"
  - "(1)"为主图，其他为详情图
  - 同类产品自动归为一组

七、缓存机制
================================================================================

缓存键值：
  - products_data           产品数据
  - products_data_version   缓存版本（控制刷新）
  - site_features           功能开关设置
  - theme                   当前主题
  - layout                  当前布局
  - product_compare         对比列表
  - browse_history          浏览历史
  - translations            翻译数据

版本控制：
  - v1: 初始版本
  - v2: 修复中文乱码
  - v3: 按文件名自动分组产品

八、多语言支持
================================================================================

支持语言：zh（中文）、en（英文）、ko（韩文）

数据存储：
  - 界面文本：i18n.js 中的 translations 对象
  - 产品数据：从 GitHub products.json 动态加载

语言切换：
  1. 用户点击语言按钮
  2. i18n.switchLanguage(lang)
  3. 保存到 localStorage
  4. 触发 languageChanged 事件
  5. frontend 监听事件，重新渲染产品

十一、注意事项
================================================================================

1. GitHub API 限流
   - 未认证请求每小时60次
   - 认证请求每小时5000次
   - 配置 Token 可提高限流

2. 缓存策略
   - 产品数据缓存1小时
   - 清理浏览器缓存可强制刷新

3. 图片命名
   - 使用英文/数字命名
   - 避免特殊字符
   - 按"名称 (序号)"格式命名
