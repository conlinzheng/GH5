# 项目使用指南

## 项目结构

```
GH5/
├── index.html          # 主页面
├── admin.html          # 功能设置页面
├── i18n-admin.html     # 翻译管理页面
├── css/
│   ├── style.css       # 主样式文件
│   └── config/
│       └── config-style.css  # 管理页面样式
├── js/
│   ├── frontend.js     # 核心前端逻辑
│   ├── github-api.js   # GitHub API 接口
│   ├── i18n.js         # 多语言支持
│   ├── cache-manager.js # 缓存管理
│   ├── product-modal.js # 产品详情弹窗
│   ├── product-compare.js # 产品对比
│   ├── contact-form.js  # 联系表单
│   ├── back-to-top.js   # 回到顶部
│   └── image-zoom.js    # 图片放大
├── 产品图/             # 产品图片目录
│   ├── series1/        # 系列1图片
│   └── series2/        # 系列2图片
└── README.md           # 项目说明
```

## 核心功能

### 1. 产品展示
- **产品列表**：网格布局展示产品
- **分类筛选**：按系列、价格、材质筛选
- **搜索功能**：支持商品名称搜索
- **排序功能**：按价格、名称排序
- **布局切换**：网格/列表/大图/小图模式

### 2. 产品详情
- **详情弹窗**：点击产品卡片查看详细信息
- **图片放大**：支持图片点击放大查看
- **产品对比**：最多支持4个产品对比
- **浏览历史**：记录最近浏览的产品

### 3. 多语言支持
- **语言切换**：中文、英文、韩文
- **翻译管理**：通过 `i18n-admin.html` 管理翻译文本
- **自动检测**：根据浏览器语言自动选择

### 4. 联系功能
- **联系表单**：支持Formspree和邮件客户端两种提交方式
- **社交媒体**：微信、微博、Instagram链接
- **联系信息**：邮箱、电话等联系方式

### 5. 主题与设置
- **主题切换**：明亮/暗黑/季节主题
- **功能开关**：通过 `admin.html` 管理功能开启状态
- **响应式设计**：适配桌面、平板、手机

## 配置与使用

### 1. 配置联系信息

**修改联系邮箱**：
1. 打开 `js/contact-form.js`
2. 修改 `formspreeEmail` 变量为你的邮箱地址
3. 同时修改 `mailto` 链接中的邮箱地址

**修改社交媒体链接**：
1. 打开 `index.html`
2. 在页脚部分修改社交媒体链接

### 2. 管理产品数据

**通过GitHub管理**：
1. 在GitHub仓库中创建 `products.json` 文件
2. 按照以下格式添加产品数据：

```json
{
  "products": [
    {
      "id": "1",
      "name": "产品名称",
      "name_en": "Product Name",
      "name_ko": "제품 이름",
      "price": 100,
      "description": "产品描述",
      "description_en": "Product description",
      "description_ko": "제품 설명",
      "series": "系列名称",
      "series_en": "Series Name",
      "series_ko": "시리즈 이름",
      "images": [
        {
          "filename": "image1.jpg",
          "isMain": true
        },
        {
          "filename": "image2.jpg",
          "isMain": false
        }
      ],
      "material": "材质",
      "material_en": "Material",
      "material_ko": "재질",
      "stock": 100
    }
  ]
}
```

**本地测试数据**：
- 系统会自动生成模拟数据用于本地测试
- 当GitHub API不可用时，会使用本地模拟数据

### 3. 管理功能设置

1. 打开 `admin.html` 页面
2. 使用开关开启或关闭功能
3. 点击「保存设置」按钮保存配置
4. 点击「恢复默认」按钮重置为默认配置

**可管理的功能**：
- 回到顶部按钮
- 图片放大功能
- 商品搜索
- 商品对比
- 最近浏览历史
- 商品筛选
- 布局切换
- 社交分享
- 主题切换
- 多语言支持
- 联系表单

### 4. 管理翻译文本

1. 打开 `i18n-admin.html` 页面
2. 编辑各语言的翻译文本
3. 点击「保存修改」按钮保存翻译
4. 点击「导出翻译」按钮导出翻译文件
5. 点击「导入翻译」按钮导入翻译文件
6. 点击「重置默认」按钮恢复默认翻译

**支持的语言**：
- 中文 (zh)
- 英文 (en)
- 韩文 (ko)

### 5. 部署与发布

**本地测试**：
1. 使用本地服务器打开项目文件夹
2. 推荐使用：
   - `python -m http.server 8000`
   - `npx serve`
   - 或其他静态文件服务器

**GitHub Pages部署**：
1. 将项目推送到GitHub仓库
2. 在仓库设置中开启GitHub Pages
3. 选择 `main` 分支作为源
4. 等待部署完成后访问生成的URL

**其他部署方式**：
- Netlify
- Vercel
- 任何支持静态网站的托管服务

## 技术特性

### 1. 性能优化
- **懒加载**：图片懒加载提升页面加载速度
- **缓存管理**：本地存储缓存产品数据
- **按需加载**：功能模块按需初始化
- **压缩优化**：建议部署时压缩CSS和JavaScript文件

### 2. 响应式设计
- **移动优先**：适配各种屏幕尺寸
- **断点设置**：
  - 手机：< 768px
  - 平板：768px - 1024px
  - 桌面：> 1024px
- **触摸优化**：支持触摸设备操作

### 3. 安全特性
- **CORS处理**：正确配置跨域请求
- **数据验证**：表单数据验证
- **XSS防护**：防止跨站脚本攻击

### 4. 可维护性
- **模块化设计**：功能模块化，易于维护
- **代码注释**：关键代码有详细注释
- **配置分离**：功能配置与业务逻辑分离

## 常见问题与解决方案

### 1. 图片加载失败
- **检查路径**：确保图片文件存在于 `产品图/` 目录中
- **检查文件名**：确保 `products.json` 中的图片文件名与实际文件一致
- **检查GitHub仓库**：确保图片已上传到GitHub仓库

### 2. 产品数据不显示
- **检查GitHub API**：确保GitHub仓库可访问
- **检查网络连接**：确保网络连接正常
- **检查数据格式**：确保 `products.json` 格式正确

### 3. 功能开关不生效
- **清除缓存**：清除浏览器缓存后重试
- **检查设置**：确保设置已保存
- **检查代码**：确保功能模块代码正确

### 4. 多语言切换不生效
- **检查翻译文件**：确保翻译文本已正确配置
- **检查浏览器语言**：确保浏览器语言设置正确
- **清除缓存**：清除浏览器缓存后重试

### 5. 联系表单提交失败
- **检查邮箱配置**：确保邮箱地址正确
- **检查网络连接**：确保网络连接正常
- **检查Formspree设置**：确保Formspree账户已验证

## 开发与扩展

### 1. 添加新功能
1. 在 `js/` 目录下创建新的功能模块文件
2. 在 `frontend.js` 中导入并初始化新模块
3. 在 `admin.html` 中添加功能开关
4. 在 `i18n-admin.html` 中添加相关翻译

### 2. 添加新语言
1. 在 `i18n.js` 中添加新语言支持
2. 在 `i18n-admin.html` 中添加新语言标签
3. 在翻译管理页面添加新语言的翻译文本
4. 在 `index.html` 中添加语言切换按钮

### 3. 自定义主题
1. 在 `css/style.css` 中添加新主题样式
2. 在 `js/frontend.js` 中添加主题切换逻辑
3. 在 `admin.html` 中添加主题管理选项

### 4. 扩展产品数据
1. 在 `products.json` 中添加新的产品字段
2. 在 `js/frontend.js` 中更新数据处理逻辑
3. 在 `js/product-modal.js` 中更新详情展示
4. 在 `js/product-compare.js` 中更新对比逻辑
